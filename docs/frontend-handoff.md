# 前端交接说明

这份文档是给前端实现者看的，和 `UI/GEMINI_README.md` 一致，但更偏工程角度。

## 固定配置

- 前端端口：`13000`
- 后端端口：`18080`
- API 前缀：`/api/v1`
- 开发环境默认后端地址：`http://localhost:18080`

## 设计原则

- 工具优先
- 编辑优先
- 列表优先
- 筛选优先
- 不做营销风

## 页面优先级

### P0

- 登录
- 组织/项目切换
- 翻译编辑页
- 文档列表

### P1

- 标签管理
- 历史与版本
- 导入导出任务

### P2

- 术语库
- 翻译记忆库
- 权限管理
- 平台 i18n

## 翻译编辑页建议布局

建议使用三栏或双栏增强布局。

例如：

- 左侧：文档树 / 文档列表 / 标签筛选
- 中间：条目列表
- 右侧：条目详情与编辑器

或者：

- 顶部：项目切换、全局筛选
- 左侧：文档与标签
- 主区域：条目表格
- 右侧抽屉：编辑面板

## 关键交互

- 修改首选原文语言后，条目列表和详情中的原文排序应立即变化
- 其他原文默认折叠，但展开状态在会话内可记忆
- 标签既能在文档列表里显示，也能在筛选栏中出现
- 没有权限的动作应直接禁用或隐藏，不要点击后才报错

## 接口字段建议

前端希望后端在条目接口中直接返回：

- 当前用户对这个条目的操作权限
- 文档标签
- 最后编辑信息
- 当前版本号

这样前端不需要额外拼很多次请求。
